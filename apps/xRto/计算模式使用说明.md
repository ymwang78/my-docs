# xRto 计算模式使用说明

## 概述

xRto 现在支持计算模式，允许在无窗口环境下运行计算任务和执行脚本。这对于自动化、批处理和服务器环境非常有用。

## 命令行参数

### --computeTask {taskname}
指定要执行的计算任务名称。

**支持的任务类型：**
- `simulation` - 流程仿真
- `optimization` - 流程优化  
- `calibration` - 流程整定
- 其他任务名称 - 默认使用仿真模式

### --projectFile {projectfile}
指定要加载的工程文件路径。**这是计算模式的必选参数**，系统需要知道加载哪个工程文件才能执行计算任务。

**支持的工程文件格式：**
- xRto 工程文件 (.xrto)
- 其他支持的工程文件格式

### --scriptFile {scriptfile}
指定要执行的脚本文件路径。此参数是可选的，与 `--computeTask` 一起使用。

**支持的脚本类型：**
- Python 脚本 (.py)
- Lua 脚本 (.lua)

## 使用方式

### 1. 仅执行计算任务
```bash
xRto.exe --computeTask simulation --projectFile "path/to/project.xrto"
```

### 2. 执行计算任务并运行脚本
```bash
xRto.exe --computeTask optimization --projectFile "path/to/project.xrto" --scriptFile test_compute_mode.py
```

### 3. 使用相对路径的示例
```bash
xRto.exe --computeTask simulation --projectFile "./flowsheet.xrto"
```

### 4. 查看帮助信息
```bash
xRto.exe --help
```

## 运行模式

### 计算模式（无窗口）
- 使用 `QCoreApplication`，不创建 GUI 界面
- 适用于服务器环境、自动化脚本、批处理任务
- 通过控制台输出执行状态和结果

### 正常模式（有窗口）
- 使用 `QApplication`，创建完整的 GUI 界面
- 默认模式，当没有指定 `--computeTask` 参数时启动

## 示例脚本

项目中包含了一个测试脚本 `test_compute_mode.py`，演示了如何在计算模式下执行 Python 脚本：

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
测试计算模式的Python脚本
"""

print("开始执行测试脚本...")
print("这是一个测试计算任务的脚本")

# 模拟一些计算操作
import time
import math

print("执行计算操作...")
result = 0
for i in range(10):
    result += math.sin(i) * math.cos(i)
    time.sleep(0.1)  # 模拟计算时间
    print(f"计算进度: {(i+1)*10}%")

print(f"计算完成，结果: {result}")
print("脚本执行结束")
```

## 执行流程

1. **参数解析** - 解析命令行参数，确定运行模式
2. **应用程序初始化** - 根据模式创建相应的应用程序实例
3. **系统组件初始化** - 初始化 zce、日志、Python 环境等
4. **计算模式执行**：
   - 创建流程控制器（无 UI 模式）
   - 初始化脚本控制器
   - 如果指定了脚本文件，执行脚本
   - 否则执行指定的计算任务
5. **结果输出** - 通过控制台输出执行结果
6. **资源清理** - 清理系统资源并退出

## 错误处理

程序会检查以下错误情况：
- 缺少必需的 `--computeTask` 参数
- 缺少必需的 `--projectFile` 参数
- 指定的工程文件不存在
- 指定的脚本文件不存在
- 工程文件加载失败
- 流程控制器初始化失败
- 脚本控制器初始化失败
- 脚本执行失败

所有错误信息都会输出到标准错误流（stderr），并返回相应的退出码。

## 退出码

- `0` - 成功执行
- `1` - 执行失败或参数错误

## 注意事项

1. 计算模式下不会创建任何 GUI 窗口
2. 脚本文件路径可以是相对路径或绝对路径
3. 确保 Python 环境已正确配置（../WinPy312/python）
4. 日志文件会记录详细的执行信息
5. 在服务器环境中使用时，确保所有依赖库都已安装

## 扩展功能

可以通过修改 `ComputeModeExecutor` 类来扩展计算模式的功能：
- 添加更多计算任务类型
- 支持配置文件参数
- 添加进度回调
- 实现结果文件输出
- 支持并行计算任务

## 技术实现

- **ComputeModeExecutor.h** - 计算模式执行器类定义
- **main.cpp** - 主程序入口，包含命令行参数解析和模式切换逻辑
- 使用 Qt 的信号槽机制处理异步脚本执行
- 通过 QTimer 实现非阻塞的任务启动
